# Overview
The Capital Deployment System (CDS) v1.2 is a production-grade capital commitment engine that evaluates candidate commitments under uncertainty and enforces a survival-first constitutional order. It produces deterministic, auditable decisions by binding each run to a Decision Capsule with reproducible inputs, solver settings, and hardware constraints. The system optimizes commitment sizes and execution plans only after satisfying lexicographic guardrails for survival, regulatory floors, liquidity continuity, and tail risk containment. Liquidity is modeled pathwise with endogenous fire-sale dynamics rather than by expected value alone. This spec defines the workflow, interfaces, and governance needed to ship a verifiable, production-ready decision engine. It is written to be validated via scenario-based testing and phase-gated acceptance. Why now: increased tail-risk exposure, liquidity fragility, and governance scrutiny require auditable, deterministic decisioning in capital deployment.
